## צעד 0 — אמת את התיקייה
ודא שאתה בתוך ריפו ה-Hugo (יש `hugo.yaml`, `content/`, `layouts/`, `.git`). אם לא — עצור ושאל. אל תיצור קבצים בתיקייה ריקה. (פירוט ב-CLAUDE.md.)

לפני שאתה נוגע בעיצוב: הרץ `hugo server -D` ותעד אילו קבצי `layouts/` כבר עברו override, ומה גרסת Hugo (`hugo version`). דווח לערן מה מצאת לפני שתתחיל.

## 1. מסטהד (כותרת ראשית גדולה)
**מטרה:** הכותרת הנוכחית קטנה ושמאלית. להפוך אותה למסטהד מרכזי גדול.

- כותרת ראשית גדולה וממורכזת: **"Illuminism · אילומיניזם"** ב-Frank Ruhl Libre, בזהב `#D4AF37`, גדולה (clamp רספונסיבי, נניח `clamp(28px, 6vw, 44px)`).
- תת-כותרת מתחת: "ערן אבן־חיים · פילוסופיה · מתמטיקה ומדע · תודעה" בטקסט קרם מעומעם.
- **אפקט ריחוף שערן ביקש:** הכותרת בזהב כברירת מחדל, ובמעבר עכבר מאירה ללבן (`transition: color .25s`).
- להשאיר **מקום לתמונת שער (cover)** מעל או מאחורי המסטהד — בלוק placeholder עכשיו (רקע אינדיגו כהה יותר), שערן ימלא בהמשך. אל תמציא תמונה.
- המסטהד מופיע בעמוד הבית; בעמודי פוסט — גרסה מוקטנת/לוגו בלבד בהדר, כדי לא לדחוף את התוכן למטה.

## 2. פריסה (סרגלים)
**ההקשר:** PaperMod הוא עמודה אחת ממורכזת. ערן ביקש שלוש עמודות (ניווט מימין, פוסטים באמצע, ווידג'טים משמאל).

**המלצה לפני ביצוע — הצג לערן את שתי האפשרויות ותן לו להחליט:**
- **א. שני סרגלים** (מה שביקש): ימין = אינדקס קריאה מסודר; אמצע = פוסטים; שמאל = אודות/ציטוט/קישורי וידאו. מקסימום מידע, אבל צפוף יותר ומורכב יותר במובייל.
- **ב. סרגל אחד חזק** (מומלץ): ימין = אינדקס קריאה מסודר + חיפוש; הווידג'טים השני (ציטוט/וידאו) מופיעים רק במסכים רחבים מאוד, או יורדים לפוטר. חוויית קריאה נקייה יותר.

**מימוש טכני:**
- CSS Grid על עמוד הבית. דוגמה לכיוון (להתאים למבנה PaperMod בפועל):
  ```css
  .ill-shell { display: grid; gap: 1.5rem; }
  @media (min-width: 1100px) {
    .ill-shell { grid-template-columns: 220px minmax(0,1fr) 240px; }
  }
  ```
- ב-RTL, העמודה הראשונה ב-DOM נוחתת מימין. סדר ה-DOM: ניווט → תוכן → ווידג'טים.
- מתחת ל-1100px: עמודה אחת; הסרגלים יורדים לתחתית או מתקפלים לתפריט. אל תשאיר שלוש עמודות צפופות בטלפון.
- accent borders → `border-inline-start` (מתהפך נכון ב-RTL).

**סרגל הניווט (החשוב ביותר):** רשימת פוסטים **מסודרת לפי סדר הקריאה** (לא לפי תאריך). כותרת "קרא לפי הסדר", מספור 01, 02, 03…, הפוסט הנוכחי מודגש. הסדר נקבע לפי `weight` ב-front matter (ראה למטה).

**סרגל הווידג'טים (אם נבחר):** תיבת "אודות" עם קטע פתיחה + "קרא עוד" שמוביל ל-`/about` (הטקסט מוכן בסעיף 6); תיבת ציטוט; תיבת קישורים לסרטונים (יוטיוב/אינסטגרם/פייסבוק — ערן ימלא כתובות).

## 3. קישורי עריכה לאדמין ("כפתורי עריכה")
**מטרה:** ערן רוצה לתקן טקסט בלחיצה, כאדמין.

- בכל עמוד פוסט, הוסף קישור "✏️ ערוך פוסט" אל עורך GitHub של אותו קובץ:
  `https://github.com/eraneh10-cloud/eraneh10-cloud.github.io/edit/main/{{ .File.Path }}`
  (השתמש במשתנה ה-Hugo המתאים לנתיב הקובץ — `.File.Path` — כדי שזה יהיה אוטומטי לכל פוסט.)
- הצג אותו **רק במצב אדמין**: סקריפט קצר שבודק `localStorage.getItem('illAdmin') === '1'` ומראה את הקישורים. כניסה ל-URL עם `?admin=1` מגדירה `illAdmin=1` ב-localStorage; `?admin=0` מנקה. ככה הכפתור מוסתר מקוראים, מופיע אצל ערן בלבד, בלי שום שרת.
- הוסף גם קישור "✏️ פוסט חדש" בהדר (מצב אדמין בלבד) אל `.../new/main/content/posts`.
- **תעד לערן:** מצב אדמין = להיכנס פעם אחת ל-`eraneh10-cloud.github.io/?admin=1`. מאז הכפתורים שלו.

## 4. פיצ'רים מקצועיים — לבנות עכשיו
- **ניווט "הקודם / הבא ברצף"** בתחתית כל פוסט, לפי סדר ה-`weight` (לא לפי תאריך). זה הלב של "קריאה לפי סדר".
- **חיפוש** — PaperMod כולל חיפוש מובנה (Fuse.js). הפעל אותו (`outputs: [HTML, RSS, JSON]` + עמוד `search`).
- **זמן קריאה משוער** + **תאריך** בראש כל פוסט.
- **תוכן עניינים** אוטומטי בפוסטים ארוכים (PaperMod: `ShowToc: true`).
- **תמונת שער לפוסט** (PaperMod תומך ב-`cover` ב-front matter) — אופציונלי לכל פוסט.
- **פוסט נעוץ** "התחילו כאן" בראש עמוד הבית (ראה סעיף 5).
- **עמוד "התחילו כאן"/מפת קריאה** שמסביר את סדר הקריאה ומקשר לפוסטים לפי הסדר.
- **404 מותאם** בעברית, בפלטה.
- ודא שה-**OpenGraph** מושך תמונת שער כשיש (לשיתוף יפה בפייסבוק/וואטסאפ).

## 5. פוסט נעוץ
- קובץ הפוסט `content/posts/000-welcome.md` (מצורף בנפרד) צריך להיות **ראשון תמיד** בעמוד הבית ובסרגל הניווט.
- מימוש: תן לו `weight: 1` (מספרים נמוכים = ראשונים) והצג תג "נעוץ · התחילו כאן". אם נדרש, התאם את לוגיקת המיון של רשימת הבית כך שתכבד `weight` לפני `date`. ודא ששאר הפוסטים מקבלים `weight` עולה (10, 20, 30…) לפי סדר הקריאה הרצוי.

## 6. תוכן מוכן ליצירה

### עמוד "אודות" → `content/about.md`
front matter: `title: "אודות"`, `weight` גבוה או מחוץ לרשימת הפוסטים (זה page, לא post). גוף הטקסט — **מילה במילה, אל תערוך:**

> אילומיניזם הוא עמוד פילוסופיה רציונלית בעברית העוסק בשאלות הקיום הגדולות דרך פילוסופיה רציונלית: מהו חומר? מהי תודעה? מהו זמן? למה המציאות בכלל ניתנת לניסוח מתמטי? למה בכלל יש חיים ולא 'כלום'? והאם אפשר לבנות מסגרת אחת שמחברת ביניהן – בלי לברוח לאמונה, ובלי להסתפק בתיאור חיצוני בלבד? אילומיניזם מנגיש גישה פילוסופית בשם מתמטיקה אונטולוגית ובמרכזה טענה מערערת: ההיגיון, ובראשו המתמטיקה, אינו רק כלי שמתאר את המציאות – הוא המציאות עצמה.
>
> **מה העמוד הזה איננו:**
> זו אינה דת. אין כאן דרישה להאמין – להפך: כל טענה חייבת לעמוד בבחינה לוגית.
> זה אינו ניו־אייג'. אין כאן זימונים, מיסטיקה עמומה או קיצורי דרך רוחניים. ה"תדרים" היחידים בעמוד הזה הם המוגדרים מתמטית.
> וזה גם אינו מדע קלאסי במובן הרגיל. המדע חשוב מאין כמותו, אבל הוא עוסק בעיקר ב"איך". כאן נשאל גם "למה", ונציע פרשנות פילוסופית לאותם נתונים, ניסויים ומשוואות עצמם. לעיתים הפרשנות תהיה שונה מן המקובל במדע – בלי להתיימר להחליף את המדע בתחום שבו הוא מצטיין.
>
> המטרה אחת: לבנות, שלב אחר שלב ובמחשבה סדורה, את התמונה השלמה – מהחושים, דרך ידע והיגיון, אל מתמטיקה, פיזיקה, תודעה וזמן.

הקטע ל**תיבת האודות בסרגל** (גרסה מקוצרת + "קרא עוד" → `/about`): "עמוד פילוסופיה רציונלית בעברית. מהו חומר? מהי תודעה? מהו זמן? למה המציאות ניתנת לניסוח מתמטי?…"

### בלוק "על הדיון" → בתחתית עמוד האודות (או כתיבת קבע בפוסט הנעוץ)
*כיוון שהתגובות חיות בפייסבוק, הטקסט הזה מתאים לעמוד "אודות" / לפוסט הנעוץ — מילה במילה:*

> כדי לראות את התמונה בשלמותה כדאי להתחיל מהפוסט הנעוץ ולקרוא לפי הסדר.
>
> על הדיון: שאלות, ביקורת והתנגדויות יתקבלו בברכה. תקפו כל טענה ככל שתרצו – את הטיעונים, לא את בני האדם. שאלה כנה מקבלת כאן את התשובה המושקעת ביותר, ולפני כן שווה לבדוק אם כבר נענתה בפוסט קודם. התנאי היחיד הוא כבוד: קללות, התקפות אישיות, ספאם וקישורים מסחריים יימחקו. העמוד יתקדם צעד אחר צעד, וכל פוסט חושף חתיכה קטנה מתמונה גדולה שתתבהר עם הזמן. חלק מהדברים יישמעו מוזרים בהתחלה – וזה בסדר; רעיונות חדשים כמעט תמיד נשמעים מוזרים לפני שהם נעשים מובנים מאליהם. הבקשה היחידה: לא להאמין מהר מדי, ולא לדחות מהר מדי. לקרוא לאט. לפקפק. לבדוק אם הטיעון מחזיק. ולהיות מוכנים לקחת רעיון עד הקצה הלוגי שלו, גם כשאינו נוח בהתחלה.

### הפוסט הנעוץ → `content/posts/000-welcome.md`
מצורף כקובץ נפרד, מוכן. השאר את גוף הטקסט כפי שהוא.

## סדר ביצוע מומלץ
1. צעד 0 (אימות תיקייה + דיווח מצב). **עצור לאישור.**
2. פוסט נעוץ + עמוד אודות (תוכן — הכי בטוח, אפס סיכון עיצובי).
3. ניווט "הקודם/הבא ברצף" + סדר `weight`.
4. מסטהד.
5. קישורי עריכה לאדמין + מצב אדמין.
6. פריסת הסרגלים — **הצג לערן א' מול ב' לפני שתבחר.**
7. חיפוש, זמן קריאה, תוכן עניינים, 404.
בכל שלב: `hugo server -D`, להראות לערן, ורק אז push. אחרי push — לעדכן את יומן העדכונים ב-CLAUDE.md.
